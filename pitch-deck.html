Technical Architecture  ·  技术架构

📱 Mobile-First

React Native + Expo. Full native Android build. Real APK — no PWA, no web wrapper.

React Native + Expo，完整原生 Android 构建，真实 APK。

🔑 MWA v2

Mobile Wallet Adapter v2 for wallet authorization, reauthorize flow, and signAndSendTransactions.

MWA v2 实现授权、reauthorize 与交易签名，全程不离 App。

⚓ Anchor Program

Each wish is a standalone PDA account on Solana Devnet. Data fetched via deterministic PDA derivation — no third-party indexers.

每个愿望是独立的 PDA 账户，通过确定性 PDA 派生读取数据。

Biggest Challenge  ·  最大挑战

🔥 Silent Transaction Failures

Using signTransactions() + manual serialization + sendRawTransaction() failed silently in Hermes — wallet confirmed, nothing landed on-chain. Fix: delegate everything to signAndSendTransactions() + MWA reauthorize flow.

常规签名方式在 Hermes 引擎中静默失败——钱包确认，但链上无记录。改用 signAndSendTransactions() 配合 reauthorize 解决。

🔧 Borsh Without BorshCoder

Anchor's BorshCoder is incompatible with Hermes. We wrote manual byte-level Borsh deserialization from scratch to parse on-chain account data reliably.

Anchor 的 BorshCoder 与 Hermes 不兼容，需从零手写字节级 Borsh 反序列化逻辑才能正确解析链上账户数据。
